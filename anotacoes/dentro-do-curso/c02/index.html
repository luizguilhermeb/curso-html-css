COMO A INTERNET FUNCIONA ?

DADOS

Os computadores só entendom 2 coisas, 0 e 1 (dois tipos de ondas de frequência).

Cada 0 e 1 recebe o nome de bit.

8 bits formam 1 byte, 1024 bytes formam 1 kilobyte, 1024 KB formam 1 megabyte, e assim por diante.

MB é usado para informar armazenamento.

Mb é usado para informar transmissão de dados.

TRANSPORTE DE DADOS

Todos esses bits são transportados principalmente através de cabos de fibra óptica. Veja este Site.

Ao contrário do que muitos pensam os satélites ainda perdem para os cabos, e não são tão usados quanto.

Para esses cabos se interligarem com sua casa é preciso se conectar com outros tipos de sistemas; como os de telefonia e tv.

No entanto, tais sistemas tem tipos diferente de ondas. É feita uma conversão, ou modulação de uma onde analógica para uma digital por exemplo.

O aparelho que faz isso contém um modem que faz isso. O nome modem vem da função dele; MOdular e DEModular (MODEM). Porém o nome do aparelho completo é Gateway.

Geralmente os algoritmos de rede irão decidir qual a melhor rota a se fazer através dos cabos, assim como o gps escolhe o melhor caminho para chegar em algum lugar dependendo de engarrafamentos.

CLIENT-SERVER

O seu dispositivo, seja um celular ou computador é chamado de cliente. Os grande computadores que armazenam os sites, são chamados de servidores.

A Internet funciona por causa de 2 protocolos, o conjunto tcp/ip. Umas das funções do TCP/IP é indentificar nós.

Um nó é cada ponto que está conectado à rede. Quando você “se conecta” à Internet, recebe uma identificação única. Essa identificação é um ENDEREÇO IP.

Basicamente, as versão antiga e ainda usada do IP é o IPV4, que contém 32 bits (ex:123.45.67.89). A versão mais moderna é o IPV6 que contém 128 bits.

Cada site tem um nome que mascara o ip do servidor dele.Esse nome fica guardado em uma grande tabela DNS (Domain Name System).

POR QUE OS SITES APARECEM AOS POUCOS?

É enviado primeiro o arquivo HTML do site, e após são requisitadas a imagens e arquivos restantes.

Qualquer arquivo em pacote é enviado em pedacinhos através dos cabos.

NOME DO SITE

O nome que o usuário compra para seu site é chamado de Domínio; ex: youtube.com, twich.tv.O nome que vem depois do "." é chamado de TLD (top level Domain).

Existem os gTLDS, que são usados para fim genérico ou comercial, ex: (.com, .net, .online). Existem os ccTLDS, que são usados para fim nacional geralmente, ex: (.br, .ru, .us) .

O nome que vem antes do Domínio, é chamado de sub-domínio, ex: www.google.com.br, images.google.com.br .

Já o que vem antes do sub-domínio é o protocolo ultilizado na rede.Na maioria das vezes é o protocolo http e https. A diferença é que o https contém segurança SSL, pois os arquivos são enviados de forma criptografada.

HOSPEDAGEM

A hospedagem é o servidor onde você vai manter todos os aquivos do seu site. Para isso é preciso pagar um plano, seja ele mensal ou anual

COMO FUNCIONA O HTTP

O protocolo http envolve a comunicação feita entre o cliente e o servidor onde o site está hospedado.

Primeiro o cliente faz uma http request, pedindo os dados do site, o servidor faz uma http response, enviando-os

Quando se faz qualquer coisa no site, um método está sendo executado através.

Alguns exemplos:

GET - pegar arquivos do site, como um vídeo
POST - postar algo no site, como uma foto
PUT - alterar algo no site, como um nome de usuário
DELETE - Deletar algo no site, como um comentário

Códigos do https:

200 - OK
201 - Created (Algo foi postado)
404 - Not Found (Não Existe)
403 - Forbidden (Sem acesso)
401 - Unauthorized (Sem autorização)
500 - Internal Server Error (Erro)
