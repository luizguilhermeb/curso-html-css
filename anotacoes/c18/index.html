O que é Git e Github

Git é um programa local que ajuda o programador a ter várias versões do projeto e voltar para qualquer versão anterior.

Github faz com que isso seja feito na nuvem, além de ser uma rede social de programadores em que se pode ajudar a desenvolver vários projetos.

(Ao se instalar o Git, não esquecer de mudar o editor padrão para o vscode na edição Default)

email noreply

O email noreply oferecido pelo github, é usado para manter seu email privado, fazendo com que suas colabborações que são mostradas no projeto como commits e pushs, mostrem apenas seu username.

.gitignore

Para adicionar arquivos ao .gitignore e ignora-los em commits e pushs é necessário usar os seguintes padrões. Matéria usada

    Localizar pastas

  • Inclua comentários com #
  • /teste.txt Prefixar uma barra obriga a correspondência apenas com arquivos na origem do repositório.
  • pasta_teste isso irá ignorar todas as pastas com o nome descrito.
  • pasta_teste/arquivo_teste isso irá ignorar apenas o arquivo_teste que está dentro da pasta.
  • logs/ Acrescentar uma barra indica que o padrão é um diretório.
  • (ignorar foo[01].txt) foo\[01\].txt Para desconsiderar os caracteres do padrão .gitignore, você pode usar \ se tiver arquivos ou diretórios que os contenham.
  • pasta/arquivo todo caminho que esteja no padrão ao lado irá começar no início, portanto uma pasta em um subdiretório mais abaixo não irá corresponder.

    Localizar pastas **

  • **/pasta_teste asteriscos duplos representam qualquer diretório, neste exemplo iram ser ignoradas todas as pastas em todos os diretórios que tiverem o nome pasta_teste.
  • **/pasta_teste/arquivo_teste aqui irá ser ignorado o arquivo_teste.
  • pasta/**/arquivo Um asterisco duplo corresponde a todos os diretórios dentro de pasta no exemplo.

    Localizar pastas *

  • *.log Um asterisco é um curinga que corresponde a zero ou mais caracteres, ou seja, qualquer coisa que tenha no final a extensão .log
  • teste* Aqui se refere a qualquer arquivo que comece com teste, o asterisco representada qualquer coisa vinda depois.

    !(excessão) e ?(caractere-curinga)

  • !pasta1/pasta2/arquivo1 se coloca uma exclamação para dizer que tal arquivo não deve ser ignorado mesmo que esteja na lista .gitignore
  • arquivo? Um ponto de interrogação corresponde exatamente a um caractere, como arquivo8 por exemplo

    Caractere curinga (colchetes)

  • arquivo[0-9].log colchetes que correspondem a apenas uma caractere dentro do intervalo.
  • arquivo[246].log colchetes que correspondem a uma caractere dentro do conjunto(2 ou 4 ou 6). Mas não 24, pois colchetes correspondem a 1 caractere.
  • arquivo[!123] qualquer caractere, com excessão de 1, 2 e 3. 12 não é uma excessão, pois colchetes correspondem a 1 caractere.
  • arquivo[a-z].log o mesmo vale para letras

Git exclude

Você também pode definir padrões pessoais para um determinado repositório em um arquivo especial em .git/info/exclude. Esses arquivos não são atualizados nem distribuídos com seu repositório. Por exemplo, se você tiver uma configuração personalizada de .gitignore apenas para você, você pode preferir que sejam adicionadas ao .git/info/exclude para evitar um commit por acidente em seu repositório que afete a equipe.
